# triage

Move issues through a state machine. Roles + state machine: `${CLAUDE_PLUGIN_ROOT}/shared/triage-labels.md`;
layout + Status/Type semantics: `${CLAUDE_PLUGIN_ROOT}/shared/issue-tracker.md`.

**Every comment/issue posted during triage MUST start with:** `> *This was generated by AI during triage.*`

## When to fire
- User wants to create/triage an issue, review incoming bugs/requests, or prep an issue for AFK.

## Procedure

0. **Onboarding gate.** Run the step-0 behavior-config check (`${CLAUDE_PLUGIN_ROOT}/shared/onboarding-gate.md`).
1. **Interpret the request** (natural language: "show what needs attention", "look at #42", "move
   #42 to ready-for-agent", "what's ready for agents?") and act.
2. **Show what needs attention** (when asked): present buckets oldest-first — untriaged,
   `needs-triage`, `needs-info` with reporter activity since last notes. Counts + one-line summary.
3. **Triage a specific issue** (a brand-new incoming bug/request has no file yet — first scaffold
   `docs/work/<feature>/issues/NN-slug.md` from `templates/work/issue.md` at `Status: needs-triage`,
   then triage it):
   a. **Gather context** — full issue + comments + prior triage notes (don't re-ask resolved
      questions); explore code in glossary vocab respecting stances; read `.out-of-scope/*.md` and
      surface any prior rejection resembling this.
   b. **Recommend** category + state with reasoning + a brief code summary. Wait for direction.
   c. **Reproduce (bugs only)** before grilling — report repro/failed-repro/insufficient-detail
      (the last is a strong `needs-info`). A confirmed repro = a stronger brief.
   d. **Flesh out** under-specified issues by calling `think`.
   e. **Apply the outcome:** `ready-for-agent` → post an agent brief
      ([AGENT-BRIEF.md](references/AGENT-BRIEF.md)); `ready-for-human` → same structure + why it
      can't be delegated; `needs-info` → post triage notes (established-so-far + specific
      questions); `wontfix` bug → polite close; `wontfix` enhancement → write to `.out-of-scope/`
      and link ([OUT-OF-SCOPE.md](references/OUT-OF-SCOPE.md)).
4. **Quick override:** if told "move #42 to X", confirm the actions then apply directly; skip
   grilling. Run the "thought-enough" checklist before any `ready-for-agent` stamp (AFK gate).

## Pipeline
- Reads:  `docs/work/<feature>/issues/NN-slug.md`; `.out-of-scope/*`; code
- Writes: issue `Status:`/`Type:` + comments; `.out-of-scope/<concept>.md`
- Next:   execute-issue (for a ready-for-agent issue) | think (to flesh out)
